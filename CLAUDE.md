# DeepfakeBench — AIGI Detection Project

## 操作规范

1. **最小改动原则**: 代码修改必须是最小的，改动尽可能小、尽可能少。避免不必要的重构、格式化或引入额外抽象。优先使用精准的 Edit 而非重写整个文件。

2. **学术代码严格验证**: 公式和理论的实现必须反复验证，确保绝对正确。遇到任何不清楚的地方必须询问用户，不可自行猜测或假设。

3. **理论文档维护**: 项目根目录的 `THEORY.md` 记录所有用到的理论及详细讲解。每次新增理论必须同步更新该文档。

4. **变动前先告知**: 任何代码或文件的修改，必须先告知用户要改哪里、怎么改，得到确认后再动手。不可自行直接修改。
